# EVzone Platform Backend

NestJS backend for the EVzone Global Green Finance Platform. The backend is now an Nx-based modular monolith with Prisma/PostgreSQL as the source of truth, Kafka-backed durable events, Redis operational helpers, and S3-compatible object storage for documents and media.

## Architecture

```text
apps/
  api/                 REST API, JWT auth, Swagger, domain modules
  worker-events/       Outbox publisher to Kafka
  worker-media/        Upload verification/scanning/thumbnail placeholder
  worker-compliance/   Compliance workflow placeholder
  scheduler/           Scheduled platform tasks

libs/
  common/ permissions/auth/config/database/events/redis/storage/audit

prisma/
  schema.prisma
  migrations/
  seed.ts
```

## Stack

| Layer | Technology |
| --- | --- |
| API | NestJS 10 + TypeScript |
| Workspace | Nx |
| Database | PostgreSQL + Prisma |
| Events | Kafka + outbox pattern |
| Cache/ops | Redis |
| Files | S3-compatible object storage, MinIO locally |
| Auth | First-party JWT access/refresh tokens |

## Quick Start

```bash
cp .env.example .env
npm install
npm run prisma:generate
npm run prisma:migrate -- --name init
npm run seed
npm run start:dev
```

API: `http://localhost:3000/api/v1`  
Swagger: `http://localhost:3000/docs`

## Docker

```bash
cp .env.example .env
docker compose up -d
```

Compose starts PostgreSQL, Redis, Kafka, MinIO, the API, and worker apps.

## API Notes

- Public project discovery remains available under `/api/v1/projects`.
- Auth is now JWT-first: `/auth/register`, `/auth/login`, `/auth/refresh`, `/auth/logout`, `/auth/me`, and password flows.
- Project gallery and document uploads use signed upload intents backed by object storage metadata.
- `/documents/upload` remains as a compatibility multipart endpoint, but new clients should prefer signed upload intents.
- Investment commitments require an idempotency key and create balanced ledger entries.
- Durable business events are written to `OutboxEvent` rows and published by `worker-events`.

## Security Defaults

- Protected routes use JWT authentication and role checks.
- Tenant membership is checked for sensitive resource access.
- Prisma is the only ORM.
- Dynamic sorting is whitelisted in services.
- Redis is used only for short-lived operational data, not critical business-event delivery.
- Object storage keys are generated by the backend; clients receive signed URLs, not credentials.
